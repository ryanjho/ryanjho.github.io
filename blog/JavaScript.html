Javascript vs Ruby Documentation

November 15, 2015 | Coding

The switch from Ruby to Javascript for this week...

After going through a week's workload of learning Javascript and working on coding challenges, I find that there are some distinct differences between the documentation for Javascript as compared to Ruby

The first difference is that Ruby's documentation is a lot more comprehensive and user-friendly as compared to that of Javascript. Ruby Docs are a lot more cleanly structured, and is indexed very nicely, so that information can be searched and found very quickly. This helps when trying to refactor code, and looking through different methods and they are applied in an example.

On the other hand, Javascript's documentation (the most useful version at MDN - Mozilla Developer Network), is a little confusing and time-consuming to find information. However, I must say that the Javascript Documentation at MDN gives pretty good examples of how each function or method can be applied.

One parting comment that I will like to give is that Javascript seems to have a much deeper knowledge base in the public domain, as seen in the number of questions and answeres on the Stack Overflow platform. This perhaps gives good evidence that Javascript documentation is well covered outside the confines of the official documentation, and a quick search on Google can also do the trick to find out more about a particular Javascript method or function.
